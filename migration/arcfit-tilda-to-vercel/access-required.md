# Access Required — что нужно от владельца

Сводка всех открытых вопросов и доступов которые нужны для миграции. Заполнить и приложить к README.md.

## 🔴 Критичные (без них нельзя начать)

### 1. Доступ к DNS (nic.ru)
- [ ] Логин/пароль или API-токен RU-CENTER
- [ ] Или владелец сам делает изменения по нашей инструкции (см. [01-dns-records.md](01-dns-records.md))

**Что нужно делать:** заменить A-record arcfit.ru с Tilda IP на Vercel IP, добавить TXT для верификации.

### 2. Решение по дате/времени cutover
- [ ] Дата (рабочий день): _______
- [ ] Время старта (утро рекомендуется): _______
- [ ] Кто будет на связи во время миграции (admin/owner): _______
- [ ] Согласовано с контакт-центром / админами клуба о возможной паузе

### 3. Webhook URL текущей лид-формы
Сейчас на Tilda заявки идут через коннектор Tilda → куда-то. Нужно:
- [ ] Открыть Tilda админку → Сайт arcfit.ru → Forms → Connectors
- [ ] Записать тип коннектора и URL/настройки
- [ ] Решить: использовать тот же канал в Next.js или новый

**Варианты:**
- A. Make.com / n8n webhook — дать URL вида `https://hook.eu1.make.com/abc123`
- B. Telegram-бот напрямую — дать токен бота и chat_id
- C. AmoCRM/Bitrix24 — incoming webhook URL
- D. Email — отказаться от Tilda email, поставить Make.com/n8n с email-action
